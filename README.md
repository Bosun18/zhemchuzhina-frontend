# Жемчужина — Фронтенд

Сайт гостевого дома «Жемчужина», Пицунда, Абхазия.  
Фронтенд написан на React + TypeScript, взаимодействует с Laravel API.

## Функционал

**Контент и навигация**
- Главная — баннер, преимущества дома, превью номеров, информация о Пицунде
- Номера — список номеров из API с фото, описанием и удобствами
- Услуги и акции — список услуг с ценами
- Новости — лента новостей и страница отдельной статьи
- Галерея — сетка фотографий с просмотром в полноэкранном режиме (лайтбокс)
- Отзывы — список отзывов гостей со средним рейтингом и ответами администратора
- Контакты — телефон, WhatsApp, адрес, геометка с координатами и кнопкой построения маршрута в Яндекс Навигаторе, встроенная карта

**Авторизация и личный кабинет**
- Регистрация и вход с валидацией форм и обработкой серверных ошибок
- Личный кабинет с двумя вкладками:
  - «Мои бронирования» — список броней со статусами, отмена брони, оставление отзыва о проживании
  - «Профиль» — редактирование личных данных и смена пароля
- Автоматический выход из аккаунта при длительном бездействии (idle logout)

**Уведомления**
- Колокольчик в шапке с бейджем количества непрочитанных уведомлений
- Выпадающий список уведомлений с отметкой как прочитанных (по одному и все сразу) и переходом по ссылке из уведомления

**Прочее**
- Баннер согласия на использование cookie
- Плавающая кнопка для связи через WhatsApp
- Глобальная обработка ошибок: дружелюбный экран при сбое рендера (ErrorBoundary), перехват необработанных ошибок и промисов, отправка отчётов об ошибках на бэкенд в продакшне
- Страница 404 для несуществующих маршрутов
- Заготовка интеграции с Яндекс Метрикой (подключается после деплоя — нужен номер счётчика)

> Страница «Бронирование» (`/booking`) находится в разработке.

## Технологии

- **React 19** — UI-библиотека
- **TypeScript** — статическая типизация
- **Vite** — сборщик и dev-сервер
- **Tailwind CSS v4** — утилитарные CSS-стили
- **React Router v7** — клиентская маршрутизация
- **Axios** — HTTP-клиент для работы с API

## Структура проекта

```
src/
├── api/           # Axios-клиент и функции запросов к API
├── components/    # Переиспользуемые компоненты (Field, Spinner, CookieBanner,
│                  # ErrorBoundary, WhatsAppButton, NotificationBell и др.)
├── context/       # React Context (AuthContext — авторизация)
├── hooks/         # Кастомные хуки (useFetch — загрузка данных, useIdleLogout)
├── layouts/       # Layouts (MainLayout с шапкой и футером)
├── pages/         # Страницы приложения
├── types/         # TypeScript-типы
└── utils/         # Утилиты (format.ts — форматирование, errors.ts — разбор
                   # ошибок API, errorHandler.ts — глобальная обработка,
                   # metrica.ts — Яндекс Метрика)
```

## Установка и запуск

### Требования

- Node.js 18+
- npm или pnpm

### Локальный запуск

```bash
# Установить зависимости
npm install

# Создать файл переменных окружения
cp .env.example .env.local

# Запустить dev-сервер
npm run dev
```

Сайт будет доступен по адресу: http://localhost:5173

### Переменные окружения

Создай файл `.env.local` в корне проекта:

```
VITE_API_URL=http://localhost:8000/api
```

В продакшне укажи реальный адрес API:

```
VITE_API_URL=https://your-domain.ru/api
```

## Сборка для продакшна

```bash
npm run build
```

Собранные файлы появятся в папке `dist/` — их нужно разместить на сервере.
